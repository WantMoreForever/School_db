# EduManage 用户使用手册

## 1. 文档说明

本文档面向当前 `EduManage` 教学管理系统的实际使用者与演示人员，按当前仓库中已经实现的页面、接口和数据库结构整理三端功能说明。

本文档只描述当前已经存在的入口、页面和行为，不编造未实现功能。具体接口与字段可配合以下文档使用：

- `docs/api-list.md`
- `docs/permission-matrix.md`
- `docs/database-schema.md`

## 2. 登录与入口

### 2.1 登录入口

- 登录页：`/login/` 或 `/login/login.php`
- 退出登录：`/login/logout.php`

登录成功后，系统会根据当前账号是否存在于 `admin`、`teacher`、`student` 表中，自动跳转到对应角色入口：

- 管理员：`/admin/index.php`
- 教师：`/teacher/index.php`
- 学生：`/student/spa.html`

### 2.2 演示账号

以下账号来自 `tests/sql/demo_seed.sql`，适合本地部署、答辩演示和 smoke 验证：

| 角色 | 姓名 | 邮箱 | 密码 |
| --- | --- | --- | --- |
| 超级管理员 | `演示超级管理员` | `admin@school.edu` | `123456` |
| 教师 | `演示教师` | `teacher@school.edu` | `123456` |
| 学生 | `演示学生` | `student@school.edu` | `123456` |

## 3. 管理员端

### 3.1 入口与导航

- 入口：`/admin/index.php`
- 顶部导航由 `admin/header.php` 和 `admin/common.php` 统一维护

当前管理员端实际页面包括：

- 首页
- 学生管理
- 教师管理
- 课程管理
- 排课管理
- 教室管理
- 公告管理
- 院系管理
- 专业管理
- 系统日志
- 个人资料
- 管理员管理（仅超级管理员可见）

### 3.2 首页

首页用于查看管理后台总览，集中展示学生、教师、课程、公告、教室、排课、院系和专业等核心数据概览，是进入各管理模块的统一起点。

### 3.3 学生管理

页面：`/admin/student.php`

当前已实现的能力包括：

- 新增学生账号和学生资料
- 编辑学生信息
- 删除学生
- 启用或禁用用户状态
- 重置学生密码
- 上传或更新头像
- 结合导入接口做学生批量导入

相关主表：

- `user`
- `student`
- `department`
- `major`

### 3.4 教师管理

页面：`/admin/teacher.php`

当前已实现的能力包括：

- 新增教师账号和教师资料
- 编辑教师信息
- 删除教师
- 启用或禁用用户状态
- 重置教师密码
- 上传或更新头像
- 结合导入接口做教师批量导入

相关主表：

- `user`
- `teacher`
- `department`

### 3.5 课程管理

页面：`/admin/course.php`

用于维护课程主数据，包括：

- 课程名称
- 学分
- 学时
- 课程说明

对应表：`course`

### 3.6 排课管理

页面：`/admin/schedule_manage.php`

这是管理员端最核心的业务页之一，当前页面和接口会同时处理：

- 为课程创建或维护开课班级 `section`
- 绑定任课教师 `teaching`
- 维护上课时间和教室 `schedule`
- 检查教师时间冲突
- 检查教室占用冲突

排课时会依赖以下真实结构：

- `section`
- `teaching`
- `schedule`
- `classroom`
- `time_slot`

### 3.7 教室管理

页面：`/admin/classroom.php`

用于维护教室资源，包括：

- 教学楼
- 房间号
- 容量
- 教室类型

对应表：`classroom`

### 3.8 公告管理

页面：`/admin/announcement.php`

当前已实现的能力包括：

- 新增公告
- 编辑公告
- 删除公告
- 设置或取消优先显示
- 查看公告详情及投放目标

管理员公告可按以下目标范围投放：

- `all`
- `students`
- `teachers`
- `major`
- `section`

对应表：

- `announcement`
- `announcement_target`

### 3.9 院系管理与专业管理

页面：

- `/admin/department.php`
- `/admin/major.php`

用于维护组织结构主数据，供教师、学生归属和限选规则使用。对应表为：

- `department`
- `major`

### 3.10 系统日志

页面：`/admin/syslog.php`

用于查看关键写操作日志，帮助管理员审计新增、编辑、删除、状态切换、公告发布等后台行为。对应表为 `system_log`。

### 3.11 个人资料

页面：`/admin/profile.php`

管理员可在此维护自己的基础资料。

### 3.12 管理员管理

页面：`/admin/admin_manage.php`

该页面只对 `super_admin` 可见，用于维护管理员账号。普通管理员不会在导航中看到该入口。

## 4. 教师端

### 4.1 入口与导航

- 入口：`/teacher/index.php`
- 页面外壳：`teacher/index.php`
- 主界面：`teacher/index.html`

教师端当前实际可见模块包括：

- 仪表盘
- 我的课程
- 成绩管理
- 考试管理
- 课程表
- 课程管理
- 课程公告
- 考勤管理
- 批量成绩上传
- 工作量报告
- 个人设置

### 4.2 仪表盘

仪表盘用于查看当前教师的授课总览，包含：

- 当前授课班级数
- 选课学生总数
- 考试记录数量
- 综合平均分
- 我的班级列表

### 4.3 我的课程

该模块用于查看当前教师任教的班级列表，并可进一步进入单个班级详情。

真实数据主要来自：

- `teaching`
- `section`
- `course`
- `takes`

### 4.4 成绩管理

该模块用于按班级管理学生成绩，当前已实现的能力包括：

- 读取班级学生名单
- 查看测验、期中、期末等成绩
- 更新字母成绩
- 自动分配字母成绩
- 查看待录入成绩提醒

真实数据主要来自：

- `takes`
- `exam`
- `section`

### 4.5 考试管理

该模块用于教师管理考试事件和分数录入，当前已实现：

- 发布考试
- 查看已发布考试
- 查看班级成绩明细
- 新增考试成绩
- 修改考试成绩
- 删除考试记录
- 取消考试事件

真实数据主要来自：

- `exam`
- `takes`
- `section`
- `teaching`

### 4.6 课程表

该模块用于查看教师本人所有排课和按周课表。相关接口会按教师任教范围读取 `schedule`、`section` 和 `classroom` 数据。

### 4.7 课程管理

这里的“课程管理”并不是管理员端的课程主数据维护，而是教师视角下的任教与开课管理。当前已实现的能力包括：

- 查看可申请任教的课程
- 申请任教
- 查看任教概览
- 移除任教关系
- 更新开课班级信息
- 查看选课统计
- 查看导师学生

对应接口主要位于 `teacher/api/application.php`。

### 4.8 课程公告

该模块用于教师向本人任教班级发布和维护课程公告，当前已实现：

- 查看某个班级的公告
- 查看本人已发布公告
- 查看教师公告收件箱
- 向任教班级发布公告
- 编辑本人公告
- 删除本人公告
- 设置或取消优先显示

教师端公告是以班级范围为核心，而不是任意目标类型自由投放。

### 4.9 考勤管理

该模块用于按排课和周次记录学生考勤，当前已实现：

- 查询班级排课
- 读取某次排课某周考勤
- 记录单条考勤
- 批量记录考勤
- 查看学生考勤摘要
- 查看班级考勤报表

对应表：

- `attendance`
- `schedule`
- `student`

### 4.10 批量成绩上传

该模块用于批量导入考试成绩，属于教师端已实现的成绩录入方式之一，不等同于管理员端的学生/教师导入。

### 4.11 工作量报告

该模块用于按学期和班级查看教师工作量汇总与明细。对应接口位于 `teacher/api/workload.php`。

### 4.12 个人设置

教师可以在此维护：

- 个人资料
- 头像
- 登录密码

## 5. 学生端

### 5.1 入口与导航

- 入口：`/student/spa.html`
- 侧边栏由 `student/js/student_sidebar.js` 按当前 hash 路由切换

学生端当前实际可见模块包括：

- 学生门户
- 选课系统
- 成绩查询
- 课程表
- 空闲教室
- 考试安排
- 通知公告
- 个人信息
- 修改密码

### 5.2 学生门户

首页用于展示学生综合摘要，包括：

- 基本身份信息
- 所属院系与专业
- GPA
- 已修学分
- 总选课数
- 最近成绩
- 快捷功能导航

### 5.3 选课系统

该模块对应 `student/api/course_select.php`，当前已实现：

- 读取当前学期已选课程
- 读取当前学期可选课程
- 选课
- 退课
- 课程时间冲突提示

当前真实选课规则包括：

- 只能操作当前学期的开课班级
- 必须在 `section.enrollment_start` 与 `section.enrollment_end` 的开放时间内
- 已选课程不能重复选
- 超出容量不能选
- 存在 `restriction` 时必须满足专业限制
- 与已选课程时间冲突时不能选
- 退课逻辑通过删除 `takes` 记录实现

### 5.4 成绩查询

该模块用于读取学生本人各课程成绩、统计结果和学业表现，不允许查看其他学生数据。

真实数据主要来自：

- `takes`
- `exam`
- `course`
- `section`

### 5.5 课程表

该模块用于读取学生本人当前课程的排课信息，包括星期、时间、教室和周次范围。

### 5.6 空闲教室

该模块对应 `student/api/free_classroom.php`，支持按以下条件查询：

- 周次
- 星期
- 起始节次
- 结束节次
- 指定教室或全部教室

其结果基于 `schedule`、`classroom` 和 `time_slot` 计算得出。

### 5.7 考试安排

该模块用于查看学生本人已选课程相关的考试安排与成绩信息。

### 5.8 通知公告

该模块用于读取学生当前可见公告。当前可见范围来自以下真实投放类型组合：

- 全体公告
- 学生端公告
- 专业公告
- 已选班级公告

对应表：

- `announcement`
- `announcement_target`
- `takes`
- `student`

### 5.9 个人信息

该模块用于查看和维护学生个人资料，当前已实现的能力包括：

- 查看个人基本信息
- 更新手机号等资料
- 上传头像

### 5.10 修改密码

该模块用于修改学生登录密码。修改成功后，当前登录状态保持不变，下次登录需要使用新密码。

## 6. 使用提示

- 管理员、教师、学生三端都依赖 PHP Session 保持登录态。
- 管理后台写操作、学生部分写操作需要 CSRF 验证。
- 若页面无法读取数据，应优先检查数据库是否已按 `school_db_backup.sql` 和 `tests/sql/demo_seed.sql` 完成导入。
- 若需更细的接口字段和错误说明，可继续查看 `docs/api-list.md`、`docs/api-contract.md` 和 `docs/error-codes.md`。
