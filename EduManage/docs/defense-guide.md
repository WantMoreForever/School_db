# EduManage 教学管理系统答辩说明

本文档用于 `EduManage` 教学管理系统的课程设计答辩、验收演示和项目介绍。重点说明系统完成度、架构设计、数据关系、核心流程、演示账号和推荐演示路线。

## 1. 项目定位

`EduManage` 教学管理系统（英文名称：EduManage, Teaching Management System）围绕管理员、教师、学生三个角色完成教学管理闭环。

- 管理员维护基础数据、用户、课程、教室、排课、公告和系统日志。
- 教师管理任教课程、考试、成绩、考勤、课程公告和工作量。
- 学生查看个人信息、课表、选课、成绩、考试安排、空闲教室和通知公告。

当前项目主体功能已经完成，适合作为课程项目提交和现场演示。

## 2. 技术架构

```text
浏览器
  |
  |  HTML / CSS / JavaScript
  v
前端页面层
  |-- login/
  |-- admin/
  |-- teacher/
  |-- student/
  |
  |  PHP Session + HTTP API
  v
后端接口层
  |-- admin/api/
  |-- teacher/api/
  |-- student/api/
  |-- components/
  |
  |  PDO
  v
MySQL 数据库
  |-- 表
  |-- 视图
  |-- 函数
  |-- 存储过程
  |-- 触发器
```

架构特点：

- 使用 PHP + MySQL + 原生前端技术，部署简单，适合课程项目演示。
- 登录态使用 PHP Session，各端接口继续校验角色权限。
- 数据访问统一走 PDO，关键查询优先调用存储过程，缺失对象时保留部分 SQL 回退逻辑。
- 前端通过 `css/portal-common.css` 统一设计变量、按钮、表单、卡片和表格风格。

## 3. 核心 ER 关系

```text
user
  |-- admin
  |-- teacher -- teaching -- section -- course
  |      |                     |
  |      |                     |-- schedule -- classroom
  |      |                     |-- exam
  |      |                     |-- attendance
  |      |
  |      |-- advisor -- student
  |
  |-- student -- takes -- section

department -- major -- student
department -- teacher

announcement -- announcement_target
system_log
```

核心说明：

- `user` 是统一账号表，`admin`、`teacher`、`student` 是角色扩展表。
- `course` 是课程基础信息，`section` 表示某课程在某学年学期的开课班级。
- `teaching` 绑定教师和开课班级，`takes` 绑定学生和开课班级。
- `schedule` 绑定开课班级、教室、周次和时间段。
- `exam` 记录考试任务和学生成绩，`attendance` 记录考勤。
- `announcement_target` 控制公告面向全体、学生、教师、专业或课程班级的可见范围。

## 4. 核心业务流程

### 登录与分流

```text
输入邮箱和密码
  -> 校验 user.status
  -> 判断账号是否属于 admin / teacher / student
  -> 跳转到对应角色入口
  -> 后续页面和接口继续校验 Session 与角色
```

### 排课与选课

```text
管理员维护课程、教师、教室、时间段
  -> 管理员创建开课班级和排课
  -> 学生查看可选课程
  -> 系统检查容量、时间冲突、选课窗口和限制
  -> 写入 takes 表
  -> 学生课表、教师班级和成绩名单同步更新
```

### 考试与成绩

```text
教师选择任教班级
  -> 发布考试任务
  -> 系统为选课学生生成考试记录
  -> 教师录入或批量导入成绩
  -> 系统计算课程平均分、成绩分布和 GPA
  -> 学生端查询成绩
```

### 公告可见性

```text
管理员或教师发布公告
  -> 写入 announcement
  -> 写入 announcement_target
  -> 学生/教师接口按角色、专业、课程关系过滤
  -> 前端展示可见公告
```

## 6. 推荐演示路线

### 6.1 管理员端

1. 登录管理员账号。
2. 打开后台首页，展示系统统计。
3. 进入学生管理或教师管理，展示增删改查、状态切换、重置密码和批量导入入口。
4. 进入课程、教室、院系、专业管理，说明基础数据维护。
5. 进入排课管理，展示开课班级、教师、教室、周次和节次关系。
6. 发布一条公告，说明公告投放范围。
7. 打开系统日志，展示操作记录。

### 6.2 教师端

1. 登录教师账号。
2. 打开仪表盘，展示授课班级、学生数、考试数和平均分。
3. 进入我的课程，查看任教班级和学生名单。
4. 进入考试/成绩管理，展示发布考试、录入成绩、批量导入和自动分配字母成绩。
5. 进入考勤管理，展示按排课和周次记录考勤。
6. 进入课程公告，发布面向班级的公告。
7. 进入工作量报告，展示按学期统计结果。

### 6.3 学生端

1. 登录学生账号。
2. 打开学生首页，展示 GPA、学分、选课数和最近成绩。
3. 进入选课系统，展示可选课程、已选课程、容量和退课/选课操作。
4. 进入课程表，展示已选课程排课。
5. 进入成绩查询，展示成绩、GPA 和学期筛选。
6. 进入考试安排，展示考试信息。
7. 进入空闲教室，按周次、星期、节次和教室条件查询。
8. 进入通知公告，验证管理员或教师发布的公告可见。
